Aprender a criar componentes reutilizáveis é uma habilidade fundamental para qualquer desenvolvedor frontend. Em projetos Next.js, a combinação com o **Tailwind CSS** oferece uma maneira poderosa de construir interfaces de forma eficiente e consistente.

Neste guia, você aprenderá a criar um componente de botão reutilizável, explicando como ele funciona e como você pode aplicá-lo em diferentes cenários no seu projeto.

### Por que Componentes Reutilizáveis?

1.  **Consistência Visual**: Mantenha a mesma aparência e comportamento para elementos comuns, como botões, cartões e formulários, em todo o seu site.
2.  **Manutenção Simplificada**: Se precisar mudar o design de um botão, você só precisa fazer a alteração em um único lugar, e ela será refletida em todas as instâncias.
3.  **Produtividade Aumentada**: Em vez de recriar o mesmo elemento várias vezes, você pode importá-lo e usá-lo onde precisar, economizando tempo e esforço.

### Criando o Componente de Botão

Vamos começar com a criação do nosso componente. Vamos chamá-lo de **PrimaryButton.tsx** e ele será responsável por renderizar um botão de destaque.

Crie o arquivo em `app/components/PrimaryButton.tsx` e adicione o seguinte código:

```tsx
'use client';

// app/components/PrimaryButton.tsx
import Link from 'next/link';
import React from 'react';

// Define as propriedades que o componente aceita.
interface PrimaryButtonProps {
  href: string;
  children: React.ReactNode;
}

export function PrimaryButton({ href, children }: PrimaryButtonProps) {
  return (
    <Link
      href={href}
      className="bg-blue-500 text-white font-semibold py-2 px-6 rounded-lg shadow-md hover:bg-blue-600 transition-colors duration-300"
    >
      {children}
    </Link>
  );
}
```

### Analisando o Código:

```javascript
import Link from 'next/link'; 
```
O Next.js usa o componente `<Link>` para navegação interna, o que otimiza o carregamento das páginas.

```typescript
interface PrimaryButtonProps;
```
Aqui, definimos as propriedades (props) que nosso componente pode receber.

- `href`: O destino do link, como `/sobre` ou `/contato`.
- `children`: O conteúdo que será exibido dentro do botão (por exemplo, o texto "Saiba mais").

### className com Tailwind CSS
A classe define a aparência do botão:

- `bg-blue-500`: Cor de fundo azul.
- `text-white`: Cor do texto branca.
- `py-2 px-6`: Preenchimento interno (padding) vertical e horizontal.
- `rounded-lg`: Bordas arredondadas.
- `shadow-md`: Adiciona uma sombra para dar profundidade.
- `hover:bg-blue-600`: Altera a cor do botão quando o mouse passa por cima.

## Como Usar o Componente

A beleza dos componentes reutilizáveis é a simplicidade de usá-los em qualquer página do seu projeto. O processo é o mesmo para qualquer lugar: importe o componente e utilize-o como uma tag de HTML.

1. **Importando o Componente**

No topo do arquivo onde você deseja usar o botão, adicione a linha de importação. Certifique-se de que o caminho (`@/components/PrimaryButton`) está correto em relação à sua estrutura de pastas.

```javascript
import { PrimaryButton } from '@/components/PrimaryButton';
```

2. **Utilizando o Componente na sua Página**

Agora que o componente foi importado, você pode usá-lo como se fosse uma tag HTML. Basta passar as propriedades que ele espera, como `href` e `children`.

Exemplo:

```javascript
// app/page.tsx (na sua página inicial)

import { PrimaryButton } from '@/components/PrimaryButton';

export default function HomePage() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-center p-24">
      <h1 className="text-4xl font-bold mb-4">Bem-vindo ao meu Blog!</h1>
      <p className="text-lg mb-8">Descubra artigos sobre tecnologia e programação.</p>

      {/* Exemplo de uso do botão */}
      <PrimaryButton href="/blog">
        Ver Últimos Posts
      </PrimaryButton>
    </main>
  );
}
```
Neste exemplo, o `<PrimaryButton>` recebe a propriedade `href` com o valor `/blog`, direcionando o usuário para a página do blog quando clicado.

Criar componentes reutilizáveis com Next.js e Tailwind CSS é uma prática que otimiza seu fluxo de trabalho e garante a consistência do design. O `PrimaryButton.tsx` que criamos é um exemplo simples, mas poderoso, de como você pode construir interfaces de forma modular e eficiente.
